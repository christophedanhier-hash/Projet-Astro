1. RÈGLES DE BASE :
   - Langue : Français uniquement.
   - Rôle : Expert en développement Web (Three.js/Canvas) et Mobile (Android/Capacitor).
   - Environnement : Je travaille sur un Chromebook (Linux/Debian).

2. LE PROJET : "Projet Astro"
   C'est une application de planétarium interactif et d'observation astronomique.
   - Stack technique : HTML5, CSS3, JavaScript (Vanilla + Modules), Three.js (pour la 3D).
   - Architecture : 
     - Dossier `www/` : Contient tout le code source web.
     - Dossier `android/` : Projet natif généré via Capacitor.
   - Objectif : Fournir une simulation précise (formules de Kepler) et visuelle du système solaire pour l'observation amateur.

3. LES MODULES CLÉS :
   - Simulateur 2D (`Simulateur_2D.html`) : Vue "du dessus" avec orbites, astéroïdes et calculs de visibilité locale.
   - Simulateur 3D (`Simulateur_3D.html`) : Vue immersive Three.js avec textures, fond étoilé et positionnement réaliste.
   - Module Lunaire (`Simulateur_Lune_3D.html`) : Focus sur les phases de la Lune, la libration et les syzygies.
   - Simulateur Terre (`Simulateur_Terre.html`) : Visualisation des saisons, de l'inclinaison axiale et de l'éclairement solaire.
   - Guide & Observations : Pages d'aide et carnet de notes.

4. ÉTAT ACTUEL :
   Le projet est en version 3.1 (Mars 2026) - Version "Offline-First".
   - **Architecture** : 100% Autonome. Toutes les ressources (textures, librairies Three.js) sont locales dans `www/assets/` et `www/js/libs/`. Utilisation des Import Maps.
   - **Précision** : Moteur mathématique unifié dans `astro-logic.js` utilisant la méthode de Newton-Raphson pour une haute précision orbitale.
   - **Interface (UI)** : Calendrier personnalisé HTML/CSS "cross-platform", bouton "Aujourd'hui", indicateurs de chargement.
   - **Compatibilité Mobile** : Support complet du tactile. Optimisations Batterie : PixelRatio limité à 2.0, arrêt du rendu en arrière-plan, réduction charge GPU.
   - **Module Terre (v3.1)** : Rendu HD (Textures Diffuse, Normal, Specular), nuages retirés pour voir les tropiques, gestion des saisons et géolocalisation.
   - **Guides** : Numéros de version et guide utilisateur à jour (v3.1).
   - **Améliorations futures (Audit v3.1)** :
     - Refactoring de l'UI : Les éléments d'interface (calendrier personnalisé, contrôles) sont dupliqués dans les 4 fichiers HTML. Il faut les extraire dans des modules UI partagés (architecture DRY).
     - Refactoring Three.js : Le code de base (Scène, Caméra, Renderer, Resize, Garbage Collection) est répété. Créer un wrapper/constructeur centralisé (`AstroRenderer.js`).
     - Séparation des préoccupations : Extraire le CSS global dans `css/main.css` et le comportement JavaScript spécifique de chaque simulateur dans le dossier `js/modules/` (ex: `earth.js`, `moon.js`).

5. FLUX DE TRAVAIL & DÉPLOIEMENT :
   - **Environnement** : Chromebook (Dev) -> Pixel 8 Pro (Cible).
   - **Commandes Clés** :
     - `npx cap sync` : Copie `www/` vers `android/`.
     - `npx cap run android --target <ID>` : Compile et installe sur le device.
   - **Débogage** : Utilisation de `chrome://inspect` pour les logs WebView.
